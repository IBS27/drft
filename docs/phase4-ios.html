App target — drft/App/

DrftApp.swift Entry point. Configures Clerk (publishable key + drft://callback redirect), wires AuthService / ConvexService / CaptureQueue together, handles drft://capture deep links by bumping a focus counter, routes other URLs to Clerk's OAuth callback handler.
RootView.swift Routing: blank page-colored screen until Clerk loads (no sign-in flash for returning users), then SignIn or Capture. Keeps Capture on screen through auth expiry (only explicit sign-out leaves). Feeds the Convex-authenticated user id to the queue.
CaptureThoughtIntent.swift App Intent "Capture a thought" (URL-representable → drft://capture) plus an AppShortcutsProvider with Siri phrases, for Shortcuts and the Action button.

Screens — drft/Features/

Capture/CaptureView.swift The app. Wordmark (tap → settings sheet), chromeless centered 30pt-light input focused on appear, live 21:47 · UNFILED timestamp (forced 24-hour), SPEAK / ● KEEP row. Keep plays fade → lone vermilion dot → reset with one soft haptic; speak shows a pulsing LISTENING state. Refocuses on warm deep-links from widgets.
Capture/CaptureModel.swift Capture state machine. keep() trims, stops dictation, enqueues synchronously before any async work, then drives the confirmation phases. Dictation transcripts stream in verbatim; late callbacks can't corrupt a kept thought.
SignIn/SignInView.swift Wordmark, "a space for unfinished thoughts", one hairline-underlined sign in button presenting Clerk's OAuth sheet. No password UI, no feature list.
Settings/SettingsView.swift Nearly-empty sheet: email, "daily thought" time picker stored in App-Group defaults (read by phase 5), confirmation-free sign out, version footer. Hairlines, no table chrome.

Services — drft/Services/

CaptureQueue.swift The never-fail guarantee. Durable FIFO in the App Group container: one atomic JSON file per capture with an append-only journal fallback, per-owner filtering (captures can't cross accounts), oldest-first flush, delete only after the server acks, 1–60s backoff, re-flush on enqueue / foreground / network-restored / sign-in. Duplicates possible, silent loss is not.
AuthService.swift Observable Clerk state (signed-in, email). Remembers the last-known owner id through auth blips so offline captures keep working; clears it only on explicit sign-out.
ConvexService.swift Convex client bound to hidden-penguin-861 via the Clerk bridge; exposes exactly one call, capture(text:) → thoughts:capture. Publishes the authenticated user id (decoded from the JWT) for the queue.
DictationService.swift On-device SFSpeechRecognizer + AVAudioEngine streaming partial transcripts. Generation-guarded start/stop (no stale-callback races), audio-session interruption and route-change teardown, permission denial routes to system settings — never an alert.

Typography.swift Helvetica Neue styles: wordmark (15pt, 0.5em tracking, uppercase), thought (30pt light), timestamp/action labels (tracked caps) as view modifiers.
Hairline.swift · NowDot.swift · Spacing.swift 1px separator, the vermilion dot, shared spacing constants.

Widget — DrftWidget/ & vendored fork

DrftWidget/DrftWidget.swift Lock-screen circular (dot only) and rectangular (wordmark + "catch a thought"), home-screen small (dot + wordmark). No data, no auth; every tap deep-links drft://capture. Reuses Stillness tokens via shared target membership.
Vendor/clerk-convex-swift/ Local fork of Clerk's official bridge (0.1.0). Upstream requests Clerk session tokens without a JWT-template parameter, so it can never mint the convex-template token the backend requires — captures would silently stay unauthenticated. The fork requests the template on login and refresh, and syncs Clerk session changes into the Convex client. Unfork when upstream adds template support.
