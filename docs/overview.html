Product & Tech Overview

drft

A space for unfinished thoughts — capture ideas the instant they occur, then develop them later with a thinking partner.

01The Idea

Not a task manager. Not a notes app. A home for thoughts that aren't anything yet — fragments that today either get lost or rot untouched in a notes app.

Capture · iOS Widget

Effortless in

A thought occurs, you type or dictate it in seconds from the home or lock screen. No filing, no tagging, no deciding where it goes.

Develop · Web Workspace

A thinking partner

Asks useful questions, connects related thoughts, challenges assumptions, gathers context — helping each fragment mature into an opinion, project, experiment, or concept.

The bet: existing tools fail at the middle. Notes apps are graveyards; task managers force premature structure. The interesting work happens between capture and conclusion — and nothing today is designed for that incubation phase.

Success metric: "thoughts I captured actually came back to me and became clearer" — not "number of notes stored." The return loop is the hard design problem: the product must pull you back (resurfacing, questions waiting for you), not passively wait.

Experience: the moment-by-moment product — capture rules, the daily return, partner contract, thought lifecycle — is specified in the experience doc. Headline decisions: iOS is capture-only; the partner prepares questions in the background but waits; the return loop is one thought a day; thoughts have no stages — just open and resting.

Design: both clients follow the Stillness design language — one thought at a time, near-monochrome, the interface receding until the fragment is the only thing in the room. Light and dark mode specs, tokens included.

02Tech Stack

LayerChoiceWhy
Mobile Native iOS — Swift / SwiftUI WidgetKit + App Intents give the best capture widget experience; on-device dictation is free, fast, and offline.
Web Vite + React + TypeScript (SPA) Workspace lives behind a login — no SEO, no SSR needed. Convex is the backend, so Next.js adds complexity for nothing.
Routing TanStack Router Type-safe, pairs well with a Vite SPA.
Styling Tailwind + shadcn/ui Fast to build, consistent, easy to make tasteful.
Backend Convex Database, built-in vector search (related thoughts), scheduled functions (resurfacing), and actions for all LLM calls. Official Swift client.
Auth Clerk Native iOS SDK plus first-class Convex integration via JWT — one auth story across both clients.
AI GPT via Vercel AI SDK Top-tier model for interactive thinking-partner sessions; cheap model for background work (embeddings, tagging, linking). AI SDK keeps models swappable, streaming for free.
Push APNs The re-engagement loop is core product, not an add-on.
Hosting Vercel Static SPA hosting, zero friction.
Repo Bun workspaces monorepo apps/ios · apps/web · packages/backend. Bun is package manager + script runner; browser and Convex are the actual runtimes. Turborepo only when builds get slow.

Architecture in one line

iOS widget → Convex mutation (capture is dumb & instant) → async action (embed · link · generate questions) → web workspace + push close the loop

Key decisions baked in

  • packages/backend, not apps/web/convex — both clients consume it; web imports generated API types directly (@drft/backend), iOS talks to the deployment.
  • Feature folders over type folders on both clients — features/thoughts/ holds its own components and hooks rather than global piles.
  • iOS lives in the monorepo but outside the Bun workspace — no JS tooling touches it; co-located for shared context only.
  • ai/ isolated inside convex — prompts and model routing in one place, so tuning the thinking-partner never touches business logic.
  • Start flat, split later — thoughts.ts can hold everything until enrichment logic justifies its own file.

04Build Plan

Vertical slices, backend-out, web before iOS. Each phase is shippable to myself.

  • 1 · Foundation — Convex schema (thoughts, connections, questions) + capture mutation; Clerk↔Convex auth; web: Tailwind + TanStack Router + Stillness tokens, minimal capture input for dogfooding.
  • 2 · Web workspace — collection list (today / this week / earlier) and thought view, no AI yet. Validates schema + design in one pass.
  • 3 · AI layer — streaming thinking-partner sessions (Convex action, AI SDK); background enrichment on capture: embeddings → vector search, auto-linking, generated questions.
  • 4 · iOS — SwiftUI capture, WidgetKit + App Intents, dictation, Convex Swift client + Clerk.
  • 5 · Return loop — scheduled resurfacing + APNs. Last on purpose: tuning it needs weeks of real captured thoughts.
